Help

Ghost ESP Commands

  • help: Display this help message.
    Usage: help
  • scanap: Start a Wi-Fi access point (AP) scan.
    Usage: scanap [seconds]
  • scansta: Start scanning for Wi-Fi stations (hops channels).
    Usage: scansta
  • stopscan: Stop any ongoing Wi-Fi scan.
    Usage: stopscan
  • attack: Launch an attack (e.g., deauthentication attack). Supports multiple selected APs when using 'select -a 1,2,3'.
    Usage: attack -d | attack -e | attack -s
    Arguments:
    -d: Start deauth attack (supports multiple APs)
    -e: Start EAPOL logoff attack
    -s: Start SAE flood attack (ESP32-C5/C6 only)
  • list: List Wi-Fi scan results or connected stations.
    Usage: list -a | list -s | list -airtags
    Arguments:
    -a: Show access points from Wi-Fi scan
    -s: List connected stations
    -airtags: List discovered AirTags
  • beaconspam: Start beacon spam with different modes.
    Usage: beaconspam [OPTION]
    Arguments:
    -r: Start random beacon spam
    -rr: Start Rickroll beacon spam
    -l: Start AP List beacon spam
    [SSID]: Use specified SSID for beacon spam
  • stopspam: Stop ongoing beacon spam.
    Usage: stopspam
  • stopdeauth: Stop ongoing deauthentication attack.
    Usage: stopdeauth
  • select: Select access point(s), station, or AirTag by index from the scan results.
    Usage: select -a <num[,num,...]> | select -s <num> | select -airtag <num>
    Arguments:
    -a: AP selection index (supports multiple: 1,3,5)
    -s: Station selection index
    -airtag: AirTag selection index
    Examples:
    select -a 4: Select single AP at index 4
    select -a 1,3,5: Select multiple APs at indices 1, 3, and 5
  • startportal: Start an Evil Portal using a local file or the default embedded page. /mnt/ prefix is added automatically to file paths if missing.
    Usage: startportal [FilePath] [AP_SSID] [PSK]
    PSK is optional for an open network. Use 'default' as the file path for the default Evil Portal.
  • stopportal: Stop Evil Portal.
    Usage: stopportal
  • blescan: Handle BLE scanning with various modes.
    Usage: blescan [OPTION]
    Arguments:
    -f: Start 'Find the Flippers' mode
    -ds: Start BLE spam detector
    -a: Start AirTag scanner
    -r: Scan for raw BLE packets
    -s: Stop BLE scanning
  • blespam: Start BLE advertisement spam attacks.
    Usage: blespam [OPTION]
    Arguments:
    -apple: Apple device spam (AirPods, Apple TV, etc.)
    -ms: Microsoft Swift Pair spam
    -samsung: Samsung Galaxy Watch spam
    -google: Google Fast Pair spam
    -random: Random spam (cycles through all types)
    -s: Stop BLE spam
  • capture: Start a WiFi Capture (Requires SD Card or Flipper).
    Usage: capture [OPTION]
    Arguments:
    -probe: Start Capturing Probe Packets
    -beacon: Start Capturing Beacon Packets
    -deauth: Start Capturing Deauth Packets
    -raw: Start Capturing Raw Packets
    -wps: Start Capturing WPS Packets and their Auth Type
    -pwn: Start Capturing Pwnagotchi Packets
    -stop: Stops the active capture
  • connect: Connects to Specific WiFi Network and saves credentials.
    Usage: connect <SSID> [Password]
  • dialconnect: Cast a Random Youtube Video on all Smart TV's on your LAN (Requires You to Run Connect First).
    Usage: dialconnect
  • powerprinter: Print Custom Text to a Printer on your LAN (Requires You to Run Connect First).
    Usage: powerprinter <Printer IP> <Text> <FontSize> <alignment>
    Alignment options: CM = Center Middle, TL = Top Left, TR = Top Right, BR = Bottom Right, BL = Bottom Left
  • blewardriving: Start/Stop BLE wardriving with GPS logging.
    Usage: blewardriving [-s]
    Arguments:
    -s: Stop BLE wardriving
  • pineap: Start/Stop detecting WiFi Pineapples.
    Usage: pineap [-s]
    Arguments:
    -s: Stop PineAP detection
  • scanports: Scan ports on local subnet or specific IP.
    Usage: scanports local [-C/-A/start_port-end_port]
    scanports [IP] [-C/-A/start_port-end_port]
    Arguments:
    -C: Scan common ports only
    -A: Scan all ports (1-65535)
    start_port-end_port: Custom port range (e.g. 80-443)
  • congestion: Display Wi-Fi channel congestion chart.
    Usage: congestion
  • apcred: Change or reset the GhostNet AP credentials.
    Usage: apcred <ssid> <password>
    apcred -r (reset to defaults)
    Arguments:
    <ssid>: New SSID for the AP
    <password>: New password (min 8 characters)
    -r: Reset to default (GhostNet/GhostNet)
  • rgbmode: Control LED effects (rainbow, police, strobe, off).
    Usage: rgbmode <rainbow|police|strobe|off|color>
  • setrgbpins: Change RGB LED pins.
    Usage: setrgbpins <red> <green> <blue>
    Note: Use same value for all pins for single-pin LED strips
  • sd_config: Show the currently configured GPIO pins for both SDMMC and SPI modes.
    Usage: sd_config
  • sd_pins_mmc: Set GPIO pins for SDMMC mode (1 or 4 bit). Requires restart/reinit. Only effective if firmware compiled for SDMMC mode.
    Usage: sd_pins_mmc <clk> <cmd> <d0> <d1> <d2> <d3>
    Example: sd_pins_mmc 19 18 20 21 22 23
  • sd_pins_spi: Set GPIO pins for SPI mode. Requires restart/reinit. Only effective if firmware compiled for SPI mode.
    Usage: sd_pins_spi <cs> <clk> <miso> <mosi>
    Example: sd_pins_spi 5 18 19 23
  • sd_save_config: Save the current SD pin configuration (both modes) to the SD card. Requires SD card to be mounted.
    Usage: sd_save_config
  • scanall: Perform combined AP and Station scan, display results.
    Usage: scanall [seconds]
  • timezone: Set the display timezone for the clock view.
    Usage: timezone <TZ_STRING>
  • beaconadd: Add an SSID to the beacon spam list.
    Usage: beaconadd <SSID>
  • beaconremove: Remove an SSID from the beacon spam list.
    Usage: beaconremove <SSID>
  • beaconclear: Clear the beacon spam list.
    Usage: beaconclear
  • beaconshow: Show the current beacon spam list.
    Usage: beaconshow
  • beaconspamlist: Start beacon spamming using the beacon spam list.
    Usage: beaconspamlist
  • dhcpstarve: DHCP starvation flood attack.
    Usage: dhcpstarve start [threads]
    dhcpstarve stop
    dhcpstarve display
  • saeflood: SAE handshake flooding attack (ESP32-C5/C6 only).
    Usage: saeflood (requires selected WPA3 AP)
  • stopsaeflood: Stop SAE flood attack.
    Usage: stopsaeflood
  • saefloodhelp: Show detailed SAE flood attack help.
    Usage: saefloodhelp
  • setcountry: Set the Wi-Fi country code (ESP32-C5 only).
    Usage: setcountry <CC>
    Arguments:
    <CC>: Two-letter ISO country code (e.g., US, GB, JP)
  • listenprobes: Listen for and log probe requests.
    Usage: listenprobes [channel] [stop]
    Arguments:
    [channel]: Listen on specific channel (1-165), omit for channel hopping
    stop: Stop probe request listening
  • webauth: Enable/disable web authentication.
    Usage: webauth <enable|disable>
  • commdiscovery: Check discovery status (auto-starts on boot).
    Usage: commdiscovery
  • commconnect: Connect to a discovered peer ESP32.
    Usage: commconnect <peer_name>
    Example: commconnect ESP_A1B2C3
  • commsend: Send a command to connected peer ESP32.
    Usage: commsend <command> [data]
    Examples:
    commsend scanap
    commsend hello world
  • commstatus: Show communication status and connection state.
    Usage: commstatus
  • commdisconnect: Disconnect from current peer.
    Usage: commdisconnect
  • commsetpins: Change communication GPIO pins at runtime.
    Usage: commsetpins <tx_pin> <rx_pin>
    Example: commsetpins 4 5
  • ethup: Bring up Ethernet interface.
    Usage: ethup
  • ethdown: Bring down Ethernet interface.
    Usage: ethdown
  • ethinfo: Show Ethernet connection information (IP, netmask, gateway, DNS, DHCP server).
    Usage: ethinfo
  • etharp: Scan local network using ARP to discover devices.
    Usage: etharp
  • ethports: Scan TCP ports on target IP address.
    Usage: ethports [local] [all | start-end] | <IP> [all | start-end]
    Arguments:
    local: Scan local network gateway
    all: Scan all ports (1-65535)
    start-end: Custom port range (e.g. 80-443)
    Examples:
    ethports local: Scan common ports on gateway
    ethports local all: Scan all ports on gateway
    ethports 192.168.1.1 80-443: Scan ports 80-443 on specific IP
  • ethping: Ping scan local network to discover active hosts.
    Usage: ethping
  • ethdns: Perform DNS lookup (forward or reverse).
    Usage: ethdns <hostname> | reverse <ip_address>
    Examples:
    ethdns google.com: Forward DNS lookup
    ethdns reverse 8.8.8.8: Reverse DNS lookup
  • ethtrace: Traceroute to target hostname or IP address.
    Usage: ethtrace <hostname_or_ip> [max_hops]
    Arguments:
    max_hops: Maximum number of hops (default: 30, max: 64)
    Examples:
    ethtrace 8.8.8.8: Traceroute to Google DNS
    ethtrace google.com 30: Traceroute with max 30 hops
  • ethstats: Show network statistics for Ethernet interface.
    Usage: ethstats
  • ethconfig: Configure Ethernet IP settings (DHCP or static).
    Usage: ethconfig dhcp | static <ip> <netmask> <gateway> | show
    Arguments:
    dhcp: Use DHCP for automatic IP assignment
    static: Set static IP configuration
    show: Show current IP configuration
    Example: ethconfig static 192.168.1.100 255.255.255.0 192.168.1.1
  • ethmac: View or set Ethernet MAC address.
    Usage: ethmac | ethmac set <xx:xx:xx:xx:xx:xx>
    Example: ethmac set 02:00:00:00:00:01
  • ethserv: Service discovery and banner grabbing on target IP.
    Usage: ethserv [ip_address]
    Note: If no IP is provided, uses local network gateway
